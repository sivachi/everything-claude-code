# Discord Bot 設計

## 概要
レシートの画像をアップロードし、管理するための Discord Bot。

## コマンド一覧

### 1. `/receipt upload`
レシート画像をアップロードして解析する。

**オプション:**
- `image`: レシート画像（必須）
- `category`: 経費科目（任意）
- `note`: メモ（任意）

**処理フロー:**
1. 画像を Supabase Storage にアップロード
2. Gemini API で OCR 解析
3. 自動カテゴリ分類
4. 結果を Discord に表示
5. ユーザーが確認・修正

### 2. `/receipt list`
レシート一覧を表示する。

**オプション:**
- `month`: 対象月（デフォルト: 当月）
- `category`: カテゴリでフィルタ

**表示内容:**
- 日付、店舗名、金額、カテゴリ
- ページネーション対応

### 3. `/receipt stats`
統計情報を表示する。

**オプション:**
- `period`: 期間（week/month/year）

**表示内容:**
- 合計金額
- カテゴリ別内訳（円グラフ）
- 前期間との比較

### 4. `/receipt export`
データをエクスポートする。

**オプション:**
- `format`: CSV/JSON
- `month`: 対象月

### 5. `/receipt category`
カテゴリ設定を管理する。

**サブコマンド:**
- `list`: カテゴリ一覧
- `set-default`: デフォルトカテゴリ設定
- `add-rule`: 自動分類ルール追加

## インタラクション設計

### アップロード確認画面
```
📸 レシート解析完了

**店舗名:** セブンイレブン
**金額:** ¥1,234
**日付:** 2024-01-15
**カテゴリ:** 消耗品費 (90% 確信度)

✅ 確定　📝 編集　❌ キャンセル
```

### 編集モーダル
- 店舗名、金額、日付、カテゴリを修正可能
- ドロップダウンでカテゴリ選択

## エラーハンドリング

1. **画像アップロードエラー**
   - ファイルサイズ制限（10MB）
   - 対応フォーマット（JPEG, PNG）

2. **OCR エラー**
   - 画像が不鮮明
   - レシートが検出できない
   - 手動入力フォーム提供

3. **権限エラー**
   - Bot の権限不足
   - ユーザーの権限確認

## セキュリティ

1. **ユーザー認証**
   - Discord ID での識別
   - 初回利用時の登録

2. **データアクセス制御**
   - 自分のレシートのみ表示
   - 管理者機能の分離

3. **レート制限**
   - コマンド実行回数制限
   - アップロード枚数制限
