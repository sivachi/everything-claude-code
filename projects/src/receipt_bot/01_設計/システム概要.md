# レシート管理システム設計書

## システム概要

Discord からレシートをアップロードし、freee の勘定科目に合わせて分類・管理するシステム。
Gemini API を使用してレシートの OCR 解析を行い、Supabase で画像とデータを保存する。

## 技術スタック

- **Bot**: Discord.js + TypeScript
- **OCR**: Gemini API (画像解析)
- **Database**: Supabase (PostgreSQL + Storage)
- **API**: Express/Fastify + TypeScript
- **Dashboard**: Next.js + TypeScript + Tailwind CSS
- **Charts**: Chart.js / Recharts

## 主要機能

1. **レシートアップロード**
   - Discord からの画像アップロード
   - Supabase Storage への保存
   - 画像 URL の永続化

2. **OCR 解析**
   - Gemini API による画像解析
   - 店舗名、金額、日付の自動抽出
   - 構造化データへの変換

3. **勘定科目分類**
   - freee 標準の勘定科目に基づく分類
   - AI による自動分類提案
   - ユーザーによる修正・学習

4. **ダッシュボード**
   - 月次・週次・日次の支出グラフ
   - 勘定科目別の分析
   - レシート画像の閲覧
   - CSV エクスポート機能

## freee 勘定科目

主要な経費科目：
- 消耗品費
- 交通費
- 会議費
- 交際費
- 新聞図書費
- 通信費
- 水道光熱費
- 支払手数料
- 雑費

## セキュリティ

- Discord OAuth2 認証
- Supabase Row Level Security
- API キーの環境変数管理
